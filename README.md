# ToolHub - 在线工具箱 & AI 导航站

一个集成了 14 个实用工具与 24 个 AI 工具导航的全栈 Web 应用。
项目探索了 AI 辅助编程在完整软件生命周期中的工程化实践。

**在线地址：** http://8.163.23.115:8088

---

## 技术栈

| 层级 | 技术 | 说明 |
| :--- | :--- | :--- |
| 后端 | Node.js + Express | RESTful API，服务端渲染 |
| 前端 | EJS 模板 + 原生 JavaScript | 响应式布局，无重型框架 |
| 数据库 | PostgreSQL | 工具数据持久化，结构化查询 |
| 部署 | Rocky Linux 9.7 + Nginx + PM2 | 反向代理，进程守护，生产环境 |
| 版本控制 | Git / GitHub | 完整提交历史与版本管理 |

## AI 辅助开发流程（核心亮点）

本项目 90% 代码由 AI 生成，但开发流程由我主导：

1. **需求拆解：** 将产品功能拆解为 AI 可执行的原子任务。
2. **Prompt 工程：** 编写精确的上下文提示词，控制 AI 输出质量。
3. **代码审查与验收：** 阅读并审查所有 AI 生成的代码，确保逻辑正确、安全可用。
4. **Bug 定位与反馈闭环：** 发现异常 → 定位根因 → 向 AI 精确描述问题 → 验证修复结果。

**结论：** AI 是高效的执行者，但对系统原理的理解、对异常的逻辑推断能力，是保证产出质量的不可替代的人类能力。

## 开发日志（Bug 排查案例）

### 案例 1：80 端口被占用，Nginx 无法启动
- **现象：** 部署 Nginx 后启动失败，`bind() to 0.0.0.0:80 failed`
- **排查过程：** `netstat -tunlp | grep :80` → 发现 `zabbix-agent` 占用 → 分析 Zabbix 组件作用 → 修改 Zabbix 配置，释放 80 端口
- **解决：** 调整端口分配策略，Nginx 绑定 80，Zabbix 迁移至其他端口

### 案例 2：代码运行器无响应
- **现象：** 在线代码执行工具提交后无输出
- **排查过程：** 检查进程存活 → 测试 stdin 管道 → 发现子进程等待标准输入但未收到数据
- **解决：** 修复父进程向子进程传递参数的方式，确保 stdin 正确写入
